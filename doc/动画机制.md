# 女仆动画实现机制

本文档梳理了 Touhou Little Maid 模组中女仆动画的实现机制，包括架构、通信模型、数据流、关键类和方法。目的是为设计依赖它的附属模组提供参考，以便无侵入地添加女仆动画逻辑。

---

## 一、整体架构

### 1.2 两种动画系统

模组包含两套动画系统：

1. **GeckoLib 动画系统**（主流）：基于 `.animation.json` 文件的关键帧动画
2. **Hardcoded 动画系统**（旧版/特殊）：通过代码直接计算骨骼旋转

## 二、GeckoLib 动画系统

### 2.1 核心类详解

#### `AnimationManager`（单例）

位置：`com.github.tartaricacid.touhoulittlemaid.client.animation.gecko.AnimationManager`

**职责**：

- 管理所有动画状态的注册和优先级调度
- 提供多个 `predicate*` 方法，每个方法对应一个动画控制器
- 按优先级（HIGHEST → LOWEST）检查并播放符合条件的动画

**优先级常量** (`Priority` 类)：

```java
HIGHEST = 0  // 死亡、游泳、睡觉等
HIGH    = 1  // 坐姿、载具等
NORMAL  = 2  // 跳跃、受伤等
LOW     = 3  // 跑步、走路等
LOWEST  = 4  // 待机 (idle)
```

**数据结构**：

```java
ReferenceArrayList<AnimationState>[] data = new ReferenceArrayList[Priority.LOWEST + 1];
```

按优先级分桶存储 `AnimationState`。

**关键方法**：

| 方法                                      | 说明                        |
| --------------------------------------- | ------------------------- |
| `predicateMain(event)`                  | 主动画逻辑，按优先级遍历检查，返回第一个匹配的动画 |
| `predicateSwing(event)`                 | 挥手动漫                      |
| `predicateUse(event)`                   | 使用物品动画                    |
| `predicateOffhandHold(event)`           | 副手持有物品动画                  |
| `predicateMainhandHold(event)`          | 主手持有物品动画                  |
| `predicateMisc(event)`                  | 杂项动画（祈求、捡雪球等）             |
| `predicateArmor(event, slot)`           | 装备动画                      |
| `predicateMagicCastingAnimation(event)` | 魔法咏唱动画（支持附属扩展）            |
| `predicatePassengerAnimation(event)`    | 乘客动画（女仆作为载具时）             |
| `getVehicleAnimation(event)`            | 载具动画（女仆乘坐时）               |

#### `AnimationState`

位置：`com.github.tartaricacid.touhoulittlemaid.client.animation.gecko.AnimationState`

**结构**：

```java
class AnimationState {
    String animationName;                              // 动画名称（对应 JSON 中的键）
    ILoopType loopType;                                // 循环类型 (LOOP / PLAY_ONCE)
    int priority;                                      // 优先级
    BiPredicate<IMaid, AnimationEvent<?>> predicate;   // 触发条件
}
```

#### `AnimationRegister`

位置：`com.github.tartaricacid.touhoulittlemaid.client.animation.gecko.AnimationRegister`

**职责**：注册所有默认动画状态

**注册的动画列表**：

| 动画名                                                     | 优先级     | 触发条件            |
| ------------------------------------------------------- | ------- | --------------- |
| `death`                                                 | HIGHEST | 实体死亡或濒死         |
| `sleep`                                                 | HIGHEST | 姿势为 SLEEPING    |
| `swim`                                                  | HIGHEST | 正在游泳            |
| `ladder_up/stillness/down`                              | HIGHEST | 爬梯时的垂直速度        |
| `gomoku`, `bookshelf`, `computer`, `keyboard`, `picnic` | HIGH    | 坐在对应的快乐方块上      |
| `boat`                                                  | HIGH    | 乘坐船             |
| `chair`                                                 | HIGH    | 乘坐其他实体（乘客）      |
| `sit`                                                   | HIGH    | 女仆坐姿模式          |
| `swim_stand`                                            | NORMAL  | 站在水中            |
| `attacked`                                              | NORMAL  | 受伤时间 > 0        |
| `jump`                                                  | NORMAL  | 不在地面且不在水中       |
| `run`                                                   | LOW     | 在地面且冲刺中         |
| `walk`                                                  | LOW     | 在地面且移动速度 > 0.05 |
| `idle`                                                  | LOWEST  | 默认（无条件，始终匹配）    |

### 2.2 动画控制器注册

在 `GeckoMaidEntity.registerControllers()` 中注册：

```java
// 8 个预并行动画控制器 (pre_parallel_0~7)
for (int i = 0; i < 8; i++) {
    addAnimationController(new AnimationController<>(this, "pre_parallel_" + i, 0, 
        e -> manager.predicateParallel(e, "pre_parallel" + i)));
}

// 主控制器（核心，transition=2 tick）
addAnimationController(new AnimationController<>(this, "main", 2, manager::predicateMain));

// 手持物品控制器
addAnimationController(new AnimationController<>(this, "hold_offhand", 0, manager::predicateOffhandHold));
addAnimationController(new AnimationController<>(this, "hold_mainhand", 0, manager::predicateMainhandHold));

// 挥动/使用控制器
addAnimationController(new AnimationController<>(this, "swing", 2, manager::predicateSwing));
addAnimationController(new AnimationController<>(this, "use", 2, manager::predicateUse));

// 魔法咏唱控制器
addAnimationController(new AnimationController<>(this, "magic_casting", 2, manager::predicateMagicCastingAnimation));

// 杂项控制器
addAnimationController(new AnimationController<>(this, "misc", 2, manager::predicateMisc));

// 乘客控制器（女仆作为载具时）
addAnimationController(new AnimationController<>(this, "passenger", 2, manager::predicatePassengerAnimation));

// 8 个并行动画控制器 (parallel_0~7)
for (int i = 0; i < 8; i++) {
    addAnimationController(new AnimationController<>(this, "parallel_" + i, 0, 
        e -> manager.predicateParallel(e, "parallel" + i)));
}

// 装备槽位动画控制器（头盔、胸甲、护腿、靴子）
for (EquipmentSlot slot : EquipmentSlot.values()) {
    if (slot.getType() == EquipmentSlot.Type.ANIMAL_ARMOR) {
        addAnimationController(new AnimationController<>(this, slot.getName(), 0, 
            e -> manager.predicateArmor(e, slot)));
    }
}
```

### 2.3 动画文件格式

动画文件位于 `src/main/resources/assets/touhou_little_maid/animation/`：

| 文件                     | 说明                    |
| ---------------------- | --------------------- |
| `maid.animation.json`  | 女仆主动画                 |
| `tac.animation.json`   | TACZ 枪械动画             |
| `chair.animation.json` | 坐垫/椅子动画               |
| `iss.animation.json`   | 铁魔法动画                 |
| `im.animation.json`    | Immersive Melodies 动画 |

**JSON 结构示例**：

```json

    "animations": {

            "loop": "hold_on_last_frame",

                "Root": {
                    "rotation": {
                        "0.0": {"post": [0, 0, 0], "lerp_mode": "catmullrom"}
                    },
                    "position": {...},
                    "scale": {...}
                }
            }
        }
    }
}
```

### 2.4 条件动画系统（`Conditional*` 类）

用于根据手持物品动态选择动画，通过 JSON 动画名称前缀来匹配：

**条件类型**：

| 前缀格式                  | 说明                             | 示例                              |
| --------------------- | ------------------------------ | ------------------------------- |
| `swing$<id>`          | 按物品 ID 匹配（副手：`swing_offhand$`） | `swing$minecraft:diamond_sword` |
| `swing#<tag>`         | 按物品 Tag 匹配                     | `swing#minecraft:swords`        |
| `swing:<use_anim>`    | 按使用动画类型匹配                      | `swing:bow`, `swing:crossbow`   |
| `hold_mainhand$<id>`  | 持有时动画（主手）                      |                                 |
| `hold_offhand#<tag>`  | 持有时动画（副手）                      |                                 |
| `use_mainhand:<type>` | 使用动画（主手）                       |                                 |

**条件管理类**：

- `ConditionManager`：管理所有条件映射
- `ConditionalSwing`：挥动条件测试
- `ConditionalUse`：使用条件测试
- `ConditionalHold`：持有条件测试
- `ConditionalVehicle`：载具条件测试
- `ConditionalPassenger`：乘客条件测试
- `ConditionalChair`：椅子条件测试
- `ConditionArmor`：护甲条件测试

## 三、Hardcoded 动画系统

### 3.1 核心类

#### `HardcodedAnimationManger`

位置：`com.github.tartaricacid.touhoulittlemaid.client.animation.HardcodedAnimationManger`

**职责**：

- 管理所有硬编码动画（通过 `ICustomAnimation` 接口）
- 在每帧渲染时调用所有注册动画的 `setRotationAngles` 方法

**注册流程**：

```java
public static void init() {
    // 游泳动画
    addMaidAnimation(new SwimAnimation());
    // 三叉戟使用动画
    addMaidAnimation(new TridentAnimation());
    // Immersive Melodies 乐器动画
    ImmersiveMelodiesCompat.addAnimation(manager);
    // 附属扩展点
    for (ILittleMaid extension : TouhouLittleMaid.EXTENSIONS) {
        extension.addHardcodeAnimation(manager);
    }
}
```

#### `ICustomAnimation<T>`

位置：`com.github.tartaricacid.touhoulittlemaid.api.animation.ICustomAnimation`

**接口方法**：

| 方法                                            | 说明                  |
| --------------------------------------------- | ------------------- |
| `setupRotations(entity, poseStack, ...)`      | 设置全局旋转（旧版模型）        |
| `setupGeckoRotations(entity, poseStack, ...)` | 设置全局旋转（GeckoLib 模型） |
| `setRotationAngles(entity, models, ...)`      | 设置骨骼旋转（旧版模型）        |
| `setGeckoRotationAngles(entity, model, ...)`  | 设置骨骼旋转（GeckoLib 模型） |

**内部动画注册** (`InnerAnimation` 类)：

- 通过 JS 脚本路径映射到 `IAnimation` 实现
- 支持动态加载资源包中的自定义动画脚本

## 四、网络通信模型

### 4.1 动画同步消息

位置：`com.github.tartaricacid.touhoulittlemaid.network.message.MaidAnimationPackage`

**用途**：服务端通知客户端播放特定动画

**数据结构**：

```java
record MaidAnimationPackage(int maidId, int animationId)
```

**预定义的动画 ID**：

```java
NONE = 0;               // 无动画
PICK_UP_SNOWBALL = 1;   // 捡雪球动画
SWF_AIM = 2;            // SW 火器瞄准
SWF_RELOAD = 3;         // SW 火器装填
SWF_FIRE = 4;           // SW 火器射击
```

**通信流程**：

```
服务端                           客户端
  │                               │
  │─ 创建 MaidAnimationPackage ──>│
  │   (maidId, animationId)       │
  │                               │
  │                         ┌─────▼─────┐
  │                         │ handle()  │
  │                         │ 更新实体  │
  │                         │ animationId│
  │                         │ animationRecordTime│
  │                         └───────────┘
```

**使用示例**：

```java
// 服务端：发送捡雪球动画
MaidAnimationPackage msg = MaidAnimationPackage.pickUpSnowball(maid);
// 通过网络发送到客户端

// 客户端：处理消息
MaidAnimationPackage.handle(msg);
// → 设置 maid.animationId = PICK_UP_SNOWBALL
// → 设置 maid.animationRecordTime = System.currentTimeMillis()
// → 设置 maid.shouldReset = true
```

### 4.2 动画触发时机

在 `AnimationManager.predicateMisc()` 中处理：

```java
if (maid instanceof EntityMaid entityMaid) {
    if (entityMaid.animationId == MaidAnimationPackage.PICK_UP_SNOWBALL) {
        // 捡雪球动画默认 1750 毫秒
        if (System.currentTimeMillis() - entityMaid.animationRecordTime > 1750) {
            entityMaid.animationId = MaidAnimationPackage.NONE;
            entityMaid.animationRecordTime = -1L;
            return playAnimation(event, "empty", PLAY_ONCE);
        }
        return playAnimation(event, "pick_up_snowball", PLAY_ONCE);
    }
}
```

## 五、数据流模型

### 5.1 客户端渲染流程

```
┌──────────────────────────────────────────────────────────────────┐
│  1. Minecraft 渲染循环调用 GeckoEntityMaidRenderer.render()      │
└─────────────────────────┬────────────────────────────────────────┘
                          │
                          ▼
┌──────────────────────────────────────────────────────────────────┐
│  2. 获取 GeckoMaidEntity (通过 AttachmentType)                    │
│     entity.getData(GeckoMaidEntity.TYPE)                         │
└─────────────────────────┬────────────────────────────────────────┘
                          │
                          ▼
┌──────────────────────────────────────────────────────────────────┐
│  3. GeckoLib 更新动画帧 (AnimatableEntity.tick)                  │
│     → 计算当前 tick = entity.tickCount + partialTick             │
└─────────────────────────┬────────────────────────────────────────┘
                          │
                          ▼
┌──────────────────────────────────────────────────────────────────┐
│  4. AnimationController.process() 处理每个控制器                  │
│     a. 调用 animationPredicate.test(event)                       │
│     b. 根据返回的 PlayState 决定是否播放动画                      │
│     c. 填充骨骼动画队列 (BoneAnimationQueue)                      │
└─────────────────────────┬────────────────────────────────────────┘
                          │
                          ▼
┌──────────────────────────────────────────────────────────────────┐
│  5. 应用动画到模型骨骼                                            │
│     → AnimatedGeoModel 更新骨骼旋转/位置/缩放                     │
└─────────────────────────┬────────────────────────────────────────┘
                          │
                          ▼
┌──────────────────────────────────────────────────────────────────┐
│  6. 执行 Hardcoded 动画 (HardcodedAnimationManger.playGeckoMaidAnimation) │
│     → 遍历所有 ICustomAnimation，调用 setGeckoRotationAngles()   │
└─────────────────────────┬────────────────────────────────────────┘
                          │
                          ▼
┌──────────────────────────────────────────────────────────────────┐
│  7. 渲染层渲染最终模型                                            │
│     → 应用所有 Layer (GeckoLayerMaidHeld, GeckoLayerMaidBanner 等)│
└──────────────────────────────────────────────────────────────────┘
```

### 5.2 动画状态决策流程（predicateMain）

```
开始
  │
  ▼
Minecraft 暂停？──是──> 返回 PlayState.STOP
  │ 否
  ▼
遍历优先级 i = HIGHEST → LOWEST
  │
  ├─ i == HIGH (载具动画)
  │    │
  │    ├─ 检查 EntityChair → 使用 ConditionalChair
  │    ├─ 检查其他载具 → 使用 ConditionalVehicle
  │    └─ 匹配成功 → 返回对应动画
  │
  ├─ 遍历当前优先级的所有 AnimationState
  │    │
  │    ├─ 调用 state.getPredicate().test(maid, event)
  │    │
  │    └─ true → 
  │         │
  │         ├─ 检查枪支部件兼容 (GunClientUtil)
  │         ├─ 播放对应动画
  │         └─ 返回 PlayState.CONTINUE
  │
  ▼
无匹配 → 返回 PlayState.STOP
```

### 5.3 动画时间线

```
Entity Tick ──┬──> GeckoMaidEntity.currentTick 更新
              │
              ├──> AnimationController.tickOffset 计算
              │
              └──> AnimationEvent 创建
                   │
                   ├── limbSwing: 行走里程
                   ├── limbSwingAmount: 行走速度
                   ├── partialTick: 插值
                   ├── extraData: EntityModelData (headPitch, netHeadYaw)
                   │
                   ▼
              AnimationPredicate 测试
                   │
                   ▼
              AnimationController.setAnimation()
                   │
                   ├── 状态过渡: RUNNING → TRANSITIONING → RUNNING
                   ├── transitionLengthTicks: 过渡时间 (通常 2 tick)
                   │
                   ▼
              BoneAnimationQueue 填充关键帧
                   │
                   ├── rotationQueue (旋转)
                   ├── positionQueue (位置)
                   └── scaleQueue (缩放)
                        │
                        ▼
                   骨骼变换应用到模型
```

## 五、事件系统

### 5.1 客户端事件总线

所有事件通过 `NeoForge.EVENT_BUS` 或 `FMLJavaModLoadingContext.get().getModEventBus()` 注册。

**事件总线类型**：

- `Bus.MOD`：模组加载事件总线，用于初始化阶段（只执行一次）
- `Bus.GAME`：游戏事件总线，用于运行时事件（每帧/每次触发执行）

## 六、生命周期与初始化流程

### 6.1 客户端初始化流程（Bus.MOD）

```
FMLClientSetupEvent
    │
    ├─► AnimationRegister.registerAnimationState()
    │   └─► 注册所有默认动画状态到 AnimationManager
    │
    ├─► HardcodedAnimationManger.init()
    │   ├─► 注册 SwimAnimation
    │   ├─► 注册 TridentAnimation
    │   ├─► ImmersiveMelodiesCompat.addAnimation()
    │   └─► ILittleMaid.addHardcodeAnimation() (附属扩展点)
    │
    ├─► MagicCastingAnimationManager.init()
    │   └─► 注册魔法咏唱动画 Provider
    │
    └─► 其他兼容层初始化
```

**代码位置**：`client/init/ClientSetupEvent.java`

```java
@SubscribeEvent
public static void onClientSetup(FMLClientSetupEvent event) {
    event.enqueueWork(AnimationRegister::registerAnimationState);
    event.enqueueWork(HardcodedAnimationManger::init);
    event.enqueueWork(MagicCastingAnimationManager::init);
}
```

### 6.2 渲染器注册流程（Bus.MOD）

```
EntityRenderersEvent.RegisterRenderers
    │
    └─► EntityRenderers.register(EntityMaid.TYPE, EntityMaidRenderer::new)
        └─► 创建女仆渲染器实例
            ├─► 添加 Layer (LayerMaidHeldItem, LayerMaidBipedHead, etc.)
            ├─► 创建 GeckoEntityMaidRenderer
            └─► 初始化 YSM 渲染器（如果安装）
```

**代码位置**：`client/init/InitEntitiesRender.java`

### 6.3 资源加载流程

```
资源包重载
    │
    ▼

    │
    ├─► clearAllCache() - 清除模型缓存
    │
    ├─► loadDefaultAnimation()
```

**代码位置**：`client/resource/GeckoModelLoader.java`

### 6.4 实体 Tick 流程（服务端/客户端）

```
EntityMaid.tick()
    │
    ├─► [服务端] Brain.tick() - AI 行为树更新
    │
    ├─► [客户端] YsmMaidClientTickEvent (YSM 兼容)
    │
    ├─► FavorabilityManager.tick() - 好感度更新
    │
```

## 七、事件详解

### 7.1 客户端渲染事件

#### `RenderMaidEvent`（客户端）

**触发时机**：女仆渲染前，在 `EntityMaidRenderer.render()` 中触发

**用途**：

- 修改女仆模型
- 修改女仆材质
- 修改动画列表
- 取消渲染

**优先级**：

- `HIGHEST`：玩家命名女仆（`=>PlayerName` 前缀）
- `NORMAL`：加密彩蛋模型
- `LOW`：普通彩蛋模型
- `LOWEST`：复活节彩蛋模型

```java
@SubscribeEvent(priority = EventPriority.HIGHEST)
public static void onRenderPlayerNamedMaid(RenderMaidEvent event) {
    Component customName = event.getMaid().asEntity().getCustomName();
    if (customName != null && customName.getString().startsWith("=>")) {
        String playerName = customName.getString().substring(2);
        MaidModels.ModelData data = event.getModelData();
        data.setModel(PlayerMaidModels.getPlayerMaidModel(playerName));
        event.setCanceled(true); // 取消后续事件
    }
}
```

**代码位置**：`api/event/client/RenderMaidEvent.java`

#### `SpecialMaidRenderEvent`（客户端）

**触发时机**：`RenderMaidEvent` 之前处理

**用途**：

- 处理玩家命名女仆（`=>PlayerName`）
- 处理彩蛋模型（SHA1 哈希名称）
- 处理普通彩蛋模型

**代码位置**：`client/event/SpecialMaidRenderEvent.java`

#### `MaidAreaRenderEvent`（客户端）

**触发时机**：世界渲染阶段 `AFTER_BLOCK_ENTITIES`

**用途**：

- 渲染女仆工作范围（红色圆柱）
- 渲染女仆 idle 范围（绿色圆柱）
- 渲染女仆睡眠范围（蓝色圆柱）

**代码位置**：`client/event/MaidAreaRenderEvent.java`

### 7.2 客户端 GUI 事件

#### `MaidContainerGuiEvent`（客户端）

三个子事件：

| 事件        | 触发时机         | 用途         |
| --------- | ------------ | ---------- |
| `Init`    | GUI 初始化时     | 添加/移除自定义按钮 |
| `Render`  | GUI 渲染时（按钮后） | 额外渲染       |
| `Tooltip` | GUI 渲染时（最后）  | 添加 Tooltip |

**代码位置**：`api/event/client/MaidContainerGuiEvent.java`

### 7.3 动画文件加载事件

#### `DefaultGeckoAnimationEvent`（客户端）

**触发时机**：资源重载时，在默认动画文件加载完成后

**用途**：

- 向默认动画文件添加额外动画
- 修改已加载的动画

```java
@SubscribeEvent
public static void onAddAnimation(DefaultGeckoAnimationEvent event) {
    // 添加自定义动画文件
    event.addAnimation(DefaultGeckoAnimationEvent.AnimationType.MAID, 
        ResourceLocation.fromNamespaceAndPath("yourmod", "animations/maid.json"));
}
```

**动画类型**：

- `MAID` - 女仆主动画
- `TAC` - TACZ 枪械动画
- `CHAIR` - 坐垫动画
- `ISS` - 铁魔法动画
- `IM` - Immersive Melodies 动画

**代码位置**：`api/event/client/DefaultGeckoAnimationEvent.java`

### 7.4 实体转换事件

#### `ConvertMaidEvent`（通用）

**触发时机**：当 `IMaid.convert(Mob)` 调用时

**用途**：

- 允许其他模组的实体使用女仆渲染系统
- 将任意 Mob 转换为 IMaid 接口

```java
@SubscribeEvent
public static void onConvertMaid(ConvertMaidEvent event) {
    if (event.getEntity() instanceof YourCustomMob mob) {
        event.setMaid(new YourCustomMaidWrapper(mob));
    }
}
```

**代码位置**：`api/event/ConvertMaidEvent.java`

### 7.5 女仆 Tick 事件

#### `MaidTickEvent`（通用，可取消）

**触发时机**：`EntityMaid.tick()` 开始时

**用途**：

- 修改女仆 AI
- 添加自定义 Tick 逻辑
- 阻止默认 Tick 行为（通过 `setCanceled(true)`

**代码位置**：`api/event/MaidTickEvent.java`

## 八、扩展点

### 8.1 添加自定义动画状态

**方式一：使用 `AnimationRegister` 扩展**

```java
// 在客户端初始化时调用
AnimationManager manager = AnimationManager.getInstance();
manager.register(new AnimationState(
    "custom_animation",           // 动画名称（需在 JSON 中定义）
    ILoopType.EDefaultLoopTypes.LOOP,  // 循环类型
    Priority.NORMAL,               // 优先级
    (maid, event) -> {             // 触发条件
        // 返回 true 时播放动画
        return maid.hasCustomFlag();
    }
));
```

**方式二：使用事件系统**

监听 `DefaultGeckoAnimationEvent` 事件（客户端）：

```java
@EventBusSubscriber(value = Dist.CLIENT, bus = EventBusSubscriber.Bus.MOD)
public class AnimationLoader {
    @SubscribeEvent
    public static void onAddAnimation(DefaultGeckoAnimationEvent event) {
        // 添加自定义动画到默认动画文件
        event.addAnimation(DefaultGeckoAnimationEvent.AnimationType.MAID, 
            ResourceLocation.fromNamespaceAndPath("yourmod", "animations/maid.json"));
    }
}
```

```java
public class CustomMagicProvider implements IMagicCastingAnimationProvider {
    @Override
    public IMagicCastingState getMagicCastingState(IMaid maid) {
        // 判断是否应该播放咏唱动画
        if (maid.isCastingSpell()) {
            return new CustomMagicCastingState(maid);
        }
        return null;
    }

    @Override
    public AnimationBuilder getAnimationBuilder(IMaid maid, IMagicCastingState state) {
        // 根据咏唱阶段返回不同动画
        return switch (state.getCurrentPhase()) {
            case START -> new AnimationBuilder().addAnimation("magic_start");
            case CASTING -> new AnimationBuilder().addAnimation("magic_cast");
            case END -> new AnimationBuilder().addAnimation("magic_end");
            default -> null;
        };
    }

    @Override
    public int getPriority() {
        return 150; // 高于默认值 100
    }
}

MagicCastingAnimationManager.registerProvider(new CustomMagicProvider());
```

## 八、最佳实践和注意事项

### 8.1 无侵入添加动画的建议

1. **优先使用 GeckoLib 动画**：通过 JSON 文件定义动画，无需修改代码
2. **使用事件系统**：监听 `DefaultGeckoAnimationEvent` 添加动画文件
3. **使用 `AnimationManager.register()`**：添加自定义动画状态
4. **避免修改原有类**：使用扩展点和 API 接口

### 8.2 注意事项

1. **客户端专用**：所有动画类都在 `client` 包下，仅在客户端运行
2. **线程安全**：动画更新在渲染线程执行，注意不要跨线程访问
3. **资源加载时机**：动画文件在资源加载阶段读取，过早访问会导致空指针
4. **动画过渡时间**：设置 `transitionLengthTicks` 来控制动画切换平滑度
5. **优先级冲突**：高优先级动画会覆盖低优先级，合理设置优先级

### 8.3 性能考虑

- 并行动画控制器（`parallel_0~7`）可以同时播放多个动画
- `AnimationState` 的谓词应尽可能轻量
- 避免在谓词中进行复杂计算或 IO 操作
