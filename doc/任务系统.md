## 一、任务系统架构

### 1. 核心架构设计

任务系统采用了 **策略模式 + 注册制** 的设计：

#### 接口层次结构：

- **`IMaidTask`** - 任务核心接口（所有任务的基接口）
  - `IAttackTask` - 近战攻击任务
  - `IRangedAttackTask` - 远程攻击任务  
  - `IFarmTask` - 农业任务
  - `IFeedTask` - 喂食任务

#### 管理器：`TaskManager`

```java
public final class TaskManager {
    private static Map<ResourceLocation, IMaidTask> TASK_MAP;  // 任务映射
    private static List<IMaidTask> TASK_INDEX;                 // 任务索引列表
    private static IMaidTask IDLE_TASK;                        // 空闲任务（默认）
}
```

**功能特性：**

- 单例模式初始化
- 支持通过 `ResourceLocation` 快速查找任务
- 支持扩展性（其他模组可通过 `ILittleMaid.addMaidTask()` 注册新任务）
- 提供非隐藏任务过滤功能

### 4. AI 行为系统

#### 大脑注册中心 (`MaidBrain`)

负责注册和管理女仆的所有 AI 行为：

```java
public static void registerBrainGoals(Brain<EntityMaid> brain, EntityMaid maid) {
    registerSchedule(brain, maid);        // 注册日程表
    registerCoreGoals(brain);             // 核心行为
    registerPanicGoals(brain);            // 慌乱行为

    registerRideIdleGoals(brain);         // 骑乘 - 空闲
    registerRideWorkGoals(brain, maid);   // 骑乘 - 工作
    registerRideRestGoals(brain);         // 骑乘 - 休息

    registerIdleGoals(brain);             // 空闲行为
    registerWorkGoals(brain, maid);       // 工作行为
    registerRestGoals(brain);             // 休息行为
}
```

**行为活动类型：**

- `Activity.CORE` - 核心行为（游泳、攀爬、跟随主人等）
- `Activity.IDLE` - 空闲行为（乞讨、娱乐、偷吃等）
- `Activity.WORK` - 工作行为（执行具体任务）
- `Activity.REST` - 休息行为（睡觉）
- `Activity.PANIC` - 慌乱行为（逃跑）
- `Activity.RIDE_*` - 骑乘状态下的行为

## 三、Brain 类深度解析 - AI 系统的心脏

### 一、Brain 类的本质和作用

在 Minecraft 的实体 AI 架构中，**`Brain<E>` 是一个泛型类**，它是实体 AI 的"大脑"，负责：

1. **存储记忆（Memory）** - 记录实体感知的信息
2. **管理活动（Activity）** - 控制不同状态下的行为优先级
3. **调度行为（BehaviorControl）** - 执行具体的 AI 任务

---

### 二、EntityMaid 的 Brain 创建流程

#### 1. Brain 的创建入口

```java

protected Brain<?> makeBrain(Dynamic<?> dynamicIn) {
```

**关键点：**

- `brainProvider()` 提供 Brain 的工厂

**Brain.Provider 包含两个核心要素：**

##### ① 记忆类型（MemoryTypes）

```java
// MaidBrain.java 第 25-42 行
public static ImmutableList<MemoryModuleType<?>> getMemoryTypes() {
    List<MemoryModuleType<?>> defaultTypes = Lists.newArrayList(
        MemoryModuleType.PATH,                    // 路径
        MemoryModuleType.DOORS_TO_CLOSE,          // 要关闭的门
        MemoryModuleType.LOOK_TARGET,             // 注视目标
        MemoryModuleType.NEAREST_HOSTILE,         // 最近的敌对生物
        MemoryModuleType.HURT_BY,                 // 伤害来源
        MemoryModuleType.HURT_BY_ENTITY,          // 伤害来源实体
        MemoryModuleType.CANT_REACH_WALK_TARGET_SINCE, // 无法到达目标的起始时间
        MemoryModuleType.WALK_TARGET,             // 行走目标
        MemoryModuleType.ATTACK_TARGET,           // 攻击目标
        MemoryModuleType.ATTACK_COOLING_DOWN,     // 攻击冷却
        InitEntities.TARGET_POS.get(),            // 目标位置（自定义）
        InitEntities.MAID_EDIBLE_BLOCK_ACTION.get() // 可食用方块动作（自定义）
    );
    // 支持扩展模组添加额外的记忆类型
    ExtraMaidBrainManager.EXTRA_MAID_BRAINS.forEach(
        extra -> defaultTypes.addAll(extra.getExtraMemoryTypes())
    );

```

**记忆的作用：**

- 存储短期信息（如当前目标）
- 跨行为共享数据（如攻击目标可以被多个行为访问）
- 有时间限制（某些记忆会自动过期）

### 四、Brain 的工作流程

#### 1. Tick 更新循环

```java
// EntityMaid.java 第 509-528 行
@Override

            "Maid's AI taking too long! Time: {} ms, Pos: ({},{},{}), Task ID: {}, Search Range: {}",

    }

```

**Brain.tick() 的执行顺序：**

1. **传感器更新** - 扫描环境并更新记忆
2. **清理过期记忆** - 移除超时信息
3. **活动切换** - 根据日程表和活动条件切换状态
4. **行为执行** - 执行当前活动的行为链

#### 2. 活动切换机制

```java
// MaidBrain.java 第 74-87 行 - 日程注册
private static void registerSchedule(Brain<EntityMaid> brain, EntityMaid maid) {
    switch (maid.getSchedule()) {
        case ALL:
            brain.setSchedule(InitEntities.MAID_ALL_DAY_SCHEDULES.get());
            break;
        case NIGHT:
            brain.setSchedule(InitEntities.MAID_NIGHT_SHIFT_SCHEDULES.get());
            break;
        case DAY:
        default:
            brain.setSchedule(InitEntities.MAID_DAY_SHIFT_SCHEDULES.get());
            break;
    }
}
```

**日程表工作原理：**

```
时间轴 (0-24000 ticks) → Schedule → 返回当前 Activity
├─ 白天 (0-12000): WORK 或 IDLE
├─ 傍晚 (12000-13000): 过渡
└─ 夜晚 (13000-24000): REST 或 WORK(夜班)
```

### 五、行为执行机制详解

#### 行为优先级和执行逻辑

```java
// 示例：工作活动注册（MaidBrain.java 第 128-151 行）
private static void registerWorkGoals(Brain<EntityMaid> brain, EntityMaid maid) {
    Pair<Integer, BehaviorControl<? super EntityMaid>> updateActivity = 
        Pair.of(99, new MaidUpdateActivityFromSchedule());

    IMaidTask task = maid.getTask();
    // 获取任务定义的 AI 行为（优先级由任务决定）
    List<Pair<Integer, BehaviorControl<? super EntityMaid>>> pairMaidList = 
        task.createBrainTasks(maid);

    if (pairMaidList.isEmpty()) {
        pairMaidList = Lists.newArrayList(updateActivity);
    } else {
        pairMaidList.add(updateActivity);
    }

    // 附加通用行为
    pairMaidList.add(Pair.of(6, new MaidBegTask()));           // 乞讨
    pairMaidList.add(Pair.of(7, new MaidWorkMealTask()));      // 工作餐
    pairMaidList.add(Pair.of(8, new MaidStealEdibleMoveBlockTask(0.6f))); // 偷吃
    pairMaidList.add(Pair.of(20, getLookAndRandomWalk(...)));  // 随机走动

    brain.addActivity(Activity.WORK, ImmutableList.copyOf(pairMaidList));
}
```

**执行规则：**

```java
// 伪代码展示 Brain 的行为选择逻辑
for (Pair<Integer, BehaviorControl> behavior : behaviors) {
    int priority = behavior.getFirst();  // 数字越小优先级越高
    BehaviorControl task = behavior.getSecond();

    if (task.checkConditions(serverWorld, entity)) {
        // 条件满足，开始执行
        task.start(serverWorld, entity, gameTime);
        return;  // 只执行一个行为！
    }
}
```

**关键点：**

- **每个 Activity 内，同一时间只能执行一个 BehaviorControl**
- **优先级数字越小越优先执行**
- **行为需要满足 `checkConditions` 才会执行**

### 六、实际案例分析

#### 案例 1：攻击任务如何工作

```java
// TaskAttack.java 第 52-66 行
@Override
public List<Pair<Integer, BehaviorControl<? super EntityMaid>>> createBrainTasks(EntityMaid maid) {
    // 优先级 5：发现敌人并开始攻击
    BehaviorControl<EntityMaid> supplementedTask = StartAttacking.create(
        this::hasAssaultWeapon,  // 检查条件：有武器
        IAttackTask::findFirstValidAttackTarget  // 查找目标
    );

    // 优先级 5：目标无效时停止攻击
    BehaviorControl<EntityMaid> findTargetTask = StopAttackingIfTargetInvalid.create(
        target -> !hasAssaultWeapon(maid) || farAway(target, maid)
    );

    // 优先级 5：走向目标
    BehaviorControl<Mob> moveToTargetTask = SetWalkTargetFromAttackTargetIfTargetOutOfReach.create(0.6f);

    // 优先级 5：执行近战攻击
    BehaviorControl<EntityMaid> attackTargetTask = MaidMeleeAttack.create(20);

    // 优先级 5：使用盾牌防御
    MaidUseShieldTask maidUseShieldTask = new MaidUseShieldTask();

    return Lists.newArrayList(
        Pair.of(5, supplementedTask),
        Pair.of(5, findTargetTask),
        Pair.of(5, moveToTargetTask),
        Pair.of(5, attackTargetTask),
        Pair.of(5, maidUseShieldTask)
    );
}
```

**执行流程：**

```
Tick 1: 传感器检测到敌人 → NEAREST_HOSTILE 记忆更新
Tick 2: StartAttacking 检查通过 → 设置 ATTACK_TARGET 记忆
Tick 3: SetWalkTargetFromAttackTarget 启动 → 设置 WALK_TARGET 记忆
Tick 4: MoveToTargetSink 执行 → 寻路移动
Tick 5: 到达范围 → MaidMeleeAttack 执行攻击
Tick 6+: 循环执行攻击直到目标死亡或丢失
```

```java
public void refreshBrain(ServerLevel serverWorldIn) {

    brain.stopAll(serverWorldIn, this);
```

- 切换任务时（如从农业切换到攻击）
- 配置改变需要重置 AI 时

### 八、关键设计模式

#### 1. 责任链模式

每个 `BehaviorControl` 都是一环，按优先级依次检查条件

#### 2. 策略模式

不同的 Task 提供不同的行为策略组合

#### 3. 观察者模式

传感器持续观察世界变化并更新记忆

#### 4. 状态模式

Activity 作为状态机，不同状态下执行不同行为
